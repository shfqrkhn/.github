# Contributing

Thanks for improving these projects.

## Before Opening Work

- Use the live demo and the README first.
- Search existing issues before opening a duplicate.
- Keep requests specific, reproducible, and scoped to one project.
- Do not paste API keys, private data, tokens, secrets, or proprietary files.

## Issues

Open a bug report for broken behavior and a feature request for new capability. Include screenshots or recordings when the issue is visual or workflow-related.

## Pull Requests

Pull requests should be small, focused, and easy to verify. Include:

- what changed
- why it changed
- screenshots for UI changes
- commands or manual checks used
- known risks or follow-up work

Changes that increase maintenance cost without clear user value may be declined.
